# 금지사항 (돌이키기 어려움)

이 파일은 **사용자·에이전트 공통 금지 규칙**입니다.
에이전트는 요청이 아래 항목에 해당할 경우 **실행하기 전에 중단하고 사용자에게 확인**합니다.

---

## 🚫 절대 금지

### 1. 권한 매트릭스 위반

- `accountant_asst` 가 승인 없이 매출 수정·삭제
- 라우트에 `requireRole(...)` 가드 누락
- 프론트엔드 UI 에서 권한 체크 없이 버튼 노출
- `roles.ts` 와 DB `role` 컬럼 값 불일치

### 2. 미승인 알림톡 템플릿

- `templates/알림톡_목록.md` 에 없는 템플릿 ID 호출 금지
- 승인된 템플릿의 변수명을 임의로 변경 금지
- 치환 변수 오탈자 (예: `#{user_name}` 이 실제 템플릿은 `#{userName}`)
- 테스트 목적이라도 프로덕션 채널로 발송 금지

### 3. 프로덕션 데이터 직접 조작

- `wrangler d1 execute --remote` 로 원격 DB 수동 조작 금지
- 마이그레이션 파일 없이 스키마 변경 금지
- 운영 중 레코드를 SQL 로 직접 `UPDATE`/`DELETE` 금지 (반드시 앱 로직 경유)

### 4. 시크릿 하드코딩

- API 키·비밀번호·토큰을 소스 파일에 직접 기입 금지
- `.env` 를 git 에 커밋 금지 (`.gitignore` 확인)
- `console.log(env.SECRET)` 같은 로깅 금지
- 테스트 코드에 실제 비밀 키 사용 금지

### 5. Git 파괴적 액션

- `master`/`main` 에 `--force` push 금지
- `git reset --hard` 로 커밋 날림 금지 (사용자 명시 요청 시만)
- `git clean -fd` 로 추적되지 않은 파일 날림 금지
- `rm -rf .git` 금지 (사용자 명시 요청 시만)

### 6. 타입 시스템 우회

- `any` 남용
- `as any`, `as unknown as X` 남용
- `// @ts-ignore` 없이 빌드를 통과시키려 `// @ts-nocheck` 파일 전체 지정

### 7. 백로그 우회

- `BACKLOG.md` 확인 없이 50줄 이상 신규 기능 시작
- 회의록 액션 아이템을 백로그에 등록하지 않고 바로 구현

### 8. 무단 배포

- `wrangler deploy` 를 사용자 확인 없이 실행 (특히 `--env production`)
- CI/CD 파이프라인에 수동 개입 금지

---

## 🟡 반드시 사용자 확인이 필요

다음은 금지가 아니지만 **자동 실행 금지, 확인 후 진행**:

- 환불 처리 (금액 무관)
- 대량 메시지 발송 (100건 이상)
- 외부 API 키 발급·교체
- 프로덕션 DB 마이그레이션 적용
- 가격·요금 정책 변경
- 사용자 계정 삭제·병합
- 외부 SaaS 구독 변경

---

## 🟢 확인 없이 해도 되는 것

안심하고 진행 가능한 작업:

- 로컬 파일 편집 (편집 자체는 reversible)
- 테스트 작성·실행
- 타입체크·린트 실행
- 개발 환경에서 `wrangler dev` 실행
- 문서(MD) 수정
- 로컬 D1 (`--local`) 에 마이그레이션 적용
- git add·commit (push 는 제외)

---

## 위반 시 복구

만약 실수로 위반이 발생했다면:

1. 즉시 사용자에게 보고
2. 영향 범위 파악 (파일·DB·외부 서비스)
3. 복구 계획 제시 후 승인 받고 실행
4. `DECISIONS.md` 에 포스트모템 추가
