# /digest-meeting — 회의록 → 백로그

`meeting-digester` 에이전트를 호출해 회의록에서 액션 아이템을 추출합니다.

## 실행 절차

### 1. 입력 선택
다음 순서로 판단:

1. 사용자가 파일 경로를 명시했으면 그 파일
   - 예: `/digest-meeting docs/meetings/2026-04-18-report.md`
2. 명시 없으면 `docs/meetings/` 에서 **mtime 기준 가장 최근 파일**
3. 파일도 없고 인자도 없으면 사용자에게 요청
   - "회의록 파일 경로를 주시거나, 요약을 이 창에 붙여넣어 주세요."

### 2. meeting-digester 에이전트 호출
- 회의록 내용 전달
- 에이전트가 추출·제안 결과 반환

### 3. 사용자 확인
에이전트 출력 후 다음을 **반드시 물어봄**:

- "위 액션 N건을 BACKLOG.md 에 추가할까요?"
- "열린 질문은 누구에게 다시 확인할까요?"

### 4. 승인 시 반영
- `BACKLOG.md` 의 `대기` 섹션에 추가 (번호는 자동 증가)
- `MEETING-LOG.md` 인덱스에 한 줄 추가
- 반영 완료 보고

## 원칙

- **자동 반영 금지** — 항상 사용자 승인 후
- 회의록 원본을 수정하지 말 것 (메모·추측 추가 금지)
- 액션 아이템이 너무 많을 때(10건+)는 상위 5개만 우선 제안

## 예시

```
/digest-meeting
```

**동작**:
1. `docs/meetings/2026-04-18-report-deduction.md` 자동 선택
2. meeting-digester 가 분석
3. 3건의 액션 아이템 + 2건의 열린 질문 제안
4. 사용자: "좋아, 반영해"
5. BACKLOG.md 에 #6, #7, #8 추가, MEETING-LOG.md 업데이트
