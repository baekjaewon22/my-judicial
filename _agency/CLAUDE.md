# CLAUDE.md

이 프로젝트는 `_agency` 부트스트랩 키트 기반으로 구성되어 있습니다.
Claude Code는 세션 시작 시 이 파일을 자동 로드합니다.

## 사용자(대표님) 프로필 요약

- **포지션**: 1인 대표, 본인이 직접 코딩
- **동시 운영**: 마이옥션 인사이드 / 파티핏(PartyFit) / 세븐스타즈 (프로젝트마다 다름 — PROJECT.md 참조)
- **기본 스택**: Cloudflare Workers + D1 + TypeScript + Tiptap + NCP Biz Message
- **커뮤니케이션**: 한국어 응답, 구체적·실용 중심, 복붙 가능한 문구 선호
- **의사결정 성향**: 오버엔지니어링 기피, 토큰·학습 비용에 민감

## 매 세션 시작 체크리스트

아래 파일들을 (존재한다면) 먼저 스캔해 맥락을 확보합니다:

1. `PRINCIPLES.md` — 언어·스택 중립 근간 원칙 (항상 적용)
2. `PROJECT.md` — 현 프로젝트 목표·범위·이해관계자
3. `STACK.md` — 기술 스택과 제약
4. `ROLES.md` — 권한 매트릭스 (코드와 일치해야 함)
5. `CONVENTIONS.md` — 이 프로젝트 스택용 관례 (있으면)
6. `BACKLOG.md` — 미완료 요구사항
7. `DECISIONS.md` — 최근 ADR
8. `DO-NOT.md` — 금지사항
9. `MEETING-LOG.md` — 최근 회의 기록

## 에이전트 팀 운영 원칙

요청이 들어오면 **어느 에이전트에 위임할지 먼저 결정**합니다:

| 요청 유형 | 담당 |
|---|---|
| "찾아봐줘", "어떻게 되어 있는지 알려줘" | researcher |
| "계획 세워줘", "어떻게 할까" | planner |
| "설계해줘", "어떤 방식이 나을까" | architect |
| "만들어줘", "구현해줘" | implementer (계획이 없으면 planner 먼저) |
| "검토해줘", "문제 없어?" | reviewer |
| "테스트 짜줘" | tester |
| "커밋·PR·문서" | scribe |
| "회의록 정리해줘" | meeting-digester |

**메인 assistant(Claude Code 자신)** 가 총괄(오케스트레이터) 역할을 합니다. 작은 요청은 직접 처리하고, 범위가 크면 에이전트에 위임합니다.

## 공통 작업 원칙

1. **코드 작성 전**에 관련 타입 정의와 권한 매트릭스를 확인
2. **신규 기능 요청**은 `BACKLOG.md` 에서 중복·충돌 검토 후 진행
3. **회의록 도착**시 `/digest-meeting` 실행
4. **알림톡** 호출 시 승인된 템플릿 목록에 있는지 대조
5. **커밋 메시지**는 `CONVENTIONS.md` 가 있으면 그 형식, 없으면 첫 줄 요약 + 본문에 "무엇/왜"
6. **큰 변경(50줄+)** 은 계획 없이 시작하지 말 것 (`PRINCIPLES.md` #12)

## 도메인 규칙

- **권한 경계**: `ROLES.md` 의 매트릭스를 벗어난 액션은 반드시 승인 플로우
- **D1 스키마 변경**: 반드시 `DECISIONS.md` 에 ADR 기록
- **외부 API 키**: 소스 하드코딩 금지, `wrangler secret put` 로만
- **프로덕션 영향**: 사용자 명시적 승인 없이 수행 금지

## 응답 스타일

- 한국어 답변 (코드·명령·파일명은 원문 유지)
- 마크다운 테이블·파일 링크 적극 활용
- 불필요한 서론·반복 금지
- 완료 보고는 1~2문장

## 금지사항 요약

`DO-NOT.md` 전체를 따르되, 특히:
- 권한 매트릭스 위반 코드
- 미승인 알림톡 템플릿 호출
- 프로덕션 DB 수동 조작
- 시크릿 하드코딩
- master/main 에 --force push
