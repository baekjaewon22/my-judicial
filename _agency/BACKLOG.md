# 업무요구사항 백로그

> 모든 기능 요청은 **먼저 여기에 등록** → 중복·충돌 검토 → 구현.
> 회의/통화 후에는 `/digest-meeting` 으로 자동 제안 받기.

|---|---|---|---|---|---|

## 대기 (todo)

| # | 제목 | 요청자 | 우선순위 | 상태 | 비고 |

## 항목 관리 규칙

### 번호 부여
- 순차 증가 정수. 재사용·재번호 금지.
- 다음 번호: (위 모든 섹션의 최대값 + 1)

### 우선순위
- `critical` — 장애, 법적 리스크, 매출 차단
- `high` — 주요 기능, 핵심 사용자 이슈
- `medium` — 개선, 중요도 중간
- `low` — 있으면 좋음

### 상태 전이
```
todo ──→ in_progress ──→ done
  │           │
  └──→ blocked ──┘
  └──→ cancelled
```

### 충돌 검토 질문
새 항목 추가 전 확인:
1. 기존 항목과 같은 파일·테이블을 건드리는가?
2. 권한 매트릭스 변경이 엮여 있는가?
3. 더 상위 목표(PROJECT.md)에 이미 포함된 내용인가?
4. 이 항목을 해결하면 다른 항목이 자동 해결되는가?

## 예시 (초기 세팅 시 삭제)

```
| 1 | 총무보조 매출 등록 시 승인 게이트 추가 | 정민호 지사장 | high | todo | ROLES.md 매트릭스 반영 필요 |
| 2 | 카드결제 입금일 매칭 자동화 | 정민호 지사장 | medium | todo | 은행 API 조사 선행 |
| 3 | 보고서 추가 제출 시 차감 로직 | 정민호 지사장 | high | blocked | 차감 기준 정책 미확정 |
```
